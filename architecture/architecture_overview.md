# Архитектура ИИ-ассистента для подготовки коммерческих предложений

## Общее описание

Разрабатываемая система представляет собой микросервисное веб-приложение, реализующее авторский алгоритм **CAGR-AG (Context-Adaptive Generation & Ranking of Arguments)** для автоматизированного создания персонализированных коммерческих предложений. Система интегрируется с корпоративными CRM-системами, парсит коммуникации с клиентами (письма, чаты, аудиозвонки) и генерирует уникальные коммерческие предложения с динамическим ранжированием аргументов на основе исторических данных об успешных сделках.

## Архитектурный стиль

Выбрана **микросервисная архитектура**, позволяющая:
- Независимо масштабировать компоненты системы
- Использовать различные технологии для разных сервисов
- Обеспечить отказоустойчивость (отказ одного сервиса не останавливает всю систему)
- Упростить поддержку и развитие отдельных модулей

## Компоненты системы

### 1. Клиентский слой (Frontend)

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Веб-интерфейс** | React.js + TypeScript | Пользовательский интерфейс для взаимодействия с системой. Содержит формы авторизации, выбора источника данных, верификации, настройки параметров, WYSIWYG-редактор для доработки КП. |
| **API-клиент** | Axios | Взаимодействие с API Gateway. |

### 2. Шлюз и интеграция (Gateway Layer)

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **API Gateway** | FastAPI (Nginx) | Единая точка входа для всех клиентских запросов. Выполняет маршрутизацию, аутентификацию, валидацию запросов, агрегацию ответов. |
| **OAuth 2.0 / SSO** | OAuth 2.0, JWT | Аутентификация пользователей через корпоративный аккаунт. |

### 3. Серверный слой (Backend Core)

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Orchestrator** | Python FastAPI | Координатор пайплайна генерации. Управляет последовательностью вызовов между сервисами: парсинг → ранжирование → генерация → сохранение. |
| **Parser Service** | Python (BeautifulSoup, Telethon, Pandas) | Сервис извлечения данных из неструктурированных источников: ссылки на переписку, аудиофайлы, CRM-сделки. Возвращает структурированный JSON (клиент, боли, бюджет, сроки). |
| **Ranking Service** | Python (Transformers, ruBERT) | Сервис ранжирования аргументов. На основе профиля клиента и истории успешных сделок выбирает топ-аргументов для включения в КП. |
| **LLM Service** | Python (YandexGPT API) | Сервис генерации финального текста. Формирует промпт на основе контекста и аргументов, отправляет запрос к YandexGPT, возвращает сгенерированный текст. |

### 4. Слой данных (Data Layer)

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Реляционная БД** | PostgreSQL | Хранение пользователей, компаний, клиентов, готовых коммерческих предложений, метаинформации. |
| **Векторная БД** | pgvector / Qdrant | Хранение эмбеддингов успешных КП для семантического поиска и ранжирования. |
| **Data Lake** | MinIO (S3-совместимое) | Хранение сырых аудиофайлов, логов диалогов, датасетов. |
| **Кэш / Очереди** | Redis | Кэширование результатов парсинга, управление фоновыми задачами через Celery. |

### 5. DevOps и инфраструктура

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| **Контейнеризация** | Docker | Упаковка каждого сервиса в контейнер. |
| **Оркестрация** | Docker Compose (разработка), Kubernetes (production) | Запуск и управление многоконтейнерным приложением. |
| **CI/CD** | GitLab CI/CD | Автоматическая сборка, тестирование и развертывание при пушах в репозиторий. |
| **Версионирование данных** | DVC | Управление версиями датасетов и моделей. |

## Взаимодействие компонентов (Sequence Diagram)

```mermaid
sequenceDiagram
    participant User as Пользователь
    participant FE as Frontend
    participant GW as API Gateway
    participant OR as Orchestrator
    participant PS as Parser Service
    participant CRM as CRM DB
    participant RS as Ranking Service
    participant ML as ML DB (векторная)
    participant LS as LLM Service
    participant ST as Storage

    User->>FE: Нажимает "Сгенерировать"
    FE->>GW: POST /api/generate (context_id)
    GW->>OR: Запуск пайплайна
    OR->>PS: Запрос на извлечение данных
    PS->>CRM: SELECT history, profile
    CRM-->>PS: Данные сделки
    PS-->>OR: Структурированный JSON
    OR->>RS: Ранжирование аргументов
    RS->>ML: Запрос топ-аргументов
    ML-->>RS: Аргументы с весами
    RS-->>OR: Список аргументов
    OR->>LS: Промпт (контекст + аргументы)
    LS-->>OR: Сгенерированный текст
    OR->>ST: Сохранение черновика
    OR-->>GW: Готовый текст
    GW-->>FE: Ответ с результатом
    FE-->>User: Отображение КП в редакторе
